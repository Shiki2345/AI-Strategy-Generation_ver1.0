# 对话模式识别

## 1. 策略类型识别模式

### 模式A：限制特定网站访问 (specific_website)
**触发关键词**：
- "禁止访问" + 网站类型（社交媒体、视频网站、游戏网站等）
- "限制" + 网站域名
- "上班时间" + "禁止" + 网站类型
- "不允许" + "访问" + 网站名称

**示例对话**：
- "我想在上班时间禁止员工访问社交媒体网站"
- "限制员工访问抖音和微博"
- "不允许访问游戏网站"

**识别逻辑**：
1. 检查是否包含"禁止"、"限制"、"不允许"等动词
2. 检查是否指定了具体网站类型或域名
3. 如果包含"所有网站"则排除此模式

### 模式B：控制所有网站操作权限 (global_website)
**触发关键词**：
- "所有网站" + "操作权限"
- "控制" + "下载" + "打印" + "上传"
- "网站功能" + "限制"
- "全局" + "网站" + "权限"

**示例对话**：
- "控制员工对所有网站的下载权限"
- "限制所有网站的文件上传功能"
- "禁止在所有网站打印敏感内容"

**识别逻辑**：
1. 检查是否包含"所有网站"或"全局"
2. 检查是否涉及操作权限（下载、打印、上传等）
3. 如果指定了具体网站则排除此模式

## 2. 多轮确认模式

### 问题1：生效对象确认
**用户回答模式**：
- **模式1**："除了老板外的所有员工"
  - 需要澄清：老板的具体定义（CEO/管理层/创始人）
  - 需要确认：是否包括各类员工类型
- **模式2**："所有员工"
  - 需要确认：是否包括实习生/外包/远程员工
- **模式3**："产品部员工"
  - 需要确认：具体部门列表
  - 需要确认：是否排除某些岗位

**Follow-up问题模板**：
```
1. 您提到的"[排除人群]"具体指什么职位/级别？
2. 是否包括实习生/外包人员/临时工？
3. 是否包括远程办公的员工？
```

### 问题2：生效时间确认
**用户回答模式**：
- **模式1**："上班时间"或"工作时间"
  - 默认：周一到周五 9:00-18:00
  - 需要确认：是否包括午休、周末是否限制
- **模式2**：具体时间段 "周一到周五，早上9点到下午6点"
  - 直接采用用户提供的时间
  - 需要确认：时区、是否排除节假日
- **模式3**："全天24小时"
  - 需要确认：是否包括非工作时间

**Follow-up问题模板**：
```
1. 具体时间段？（如：周一至周五 X:XX-Y:YY）
2. 是否包括午休时间？
3. 周末是否需要限制？
4. 采用哪个时区？
```

### 问题3：具体限制内容确认
**specific_website模式**：
- **模式1**：提供具体域名列表
  - 示例："微博 (weibo.com)、抖音网页版 (douyin.com)"
  - 直接采用用户提供的列表
- **模式2**：提供网站类型
  - 示例："社交媒体网站"
  - 需要提供常见网站列表供用户选择
- **模式3**："完全禁止" vs "功能限制"
  - 需要确认：是禁止访问还是只限制某些功能

**global_website模式**：
- **模式1**：操作权限列表
  - 示例："禁止下载、打印、上传"
  - 需要确认：是否还有其他限制
- **模式2**：文件类型限制
  - 示例："禁止下载.exe、.zip文件"
  - 需要确认：具体文件类型列表

**Follow-up问题模板**：
```
1. 具体网站列表？（如：weibo.com, douyin.com）
2. 是完全禁止访问，还是只限制某些功能？
3. 是否需要区分办公用途和个人用途？
```

### 问题4：审批流程确认
**用户回答模式**：
- **模式1**："需要申请特殊权限"
  - 默认：CEO线上审批，最长4小时
  - 需要确认：审批人、方式、有效期
- **模式2**："不需要审批"
  - 设置为`approval_process.required: false`
- **模式3**："特殊情况例外"
  - 需要确认：例外条件、审批流程

**Follow-up问题模板**：
```
1. 如果员工因工作需要访问，是否需要申请特殊权限？
2. 如果需要审批，谁来审批？（直属主管/HR/IT部门）
3. 审批流程是线上申请还是线下审批？
4. 最长有效期是多长时间？
```

## 3. 对话模式示例

### 完整示例1：工作时间社交媒体限制
```
用户：我想在上班时间禁止员工访问社交媒体网站
助手：生效对象是谁？
用户：除了老板外的所有员工
助手：什么时间生效？
用户：上班时间：周一到周五，早上9点到下午6点（标准工作时间）
助手：具体限制内容是什么？
用户：微博、抖音网页版、小红书、Facebook、Twitter/X、Instagram
助手：是否需要审批流程？
用户：如果员工在工作时间因工作需要访问这些社交媒体网站，需要申请特殊权限，由BOSS线上审批
```

**模式识别**：
- 策略类型：specific_website
- 生效对象：除CEO外的所有员工
- 生效时间：周一到周五 9:00-18:00
- 限制内容：6个社交媒体网站完全禁止
- 审批流程：CEO线上审批，工作需要时申请

### 完整示例2：全局下载限制
```
用户：控制员工对所有网站的下载权限
助手：生效对象是谁？
用户：所有员工
助手：什么时间生效？
用户：全天24小时
助手：具体限制内容是什么？
用户：禁止下载任何文件，但允许查看
助手：是否需要审批流程？
用户：工作需要下载时，由直属主管审批
```

**模式识别**：
- 策略类型：global_website
- 生效对象：所有员工
- 生效时间：全天24小时
- 限制内容：禁止下载，允许查看
- 审批流程：直属主管审批

## 4. 模式匹配算法

### 关键词权重
1. **策略类型关键词**（权重：高）
   - "禁止访问"：+3
   - "限制"：+2  
   - "控制"：+2
   - "所有网站"：+3（global_website）
   - 具体域名：+2（specific_website）

2. **时间关键词**（权重：中）
   - "上班时间"：+2
   - "工作时间"：+2
   - "全天"：+2
   - 具体时间段：+3

3. **对象关键词**（权重：中）
   - "员工"：+2
   - "除了...外"：+3
   - 部门名称：+2

4. **审批关键词**（权重：低）
   - "申请"：+1
   - "审批"：+1
   - "特殊权限"：+2

### 决策规则
1. 如果"所有网站"权重 > 3 → `global_website`
2. 如果具体域名权重 > 2 → `specific_website`
3. 如果两者权重接近 → 询问用户澄清
4. 默认策略类型：`specific_website`
