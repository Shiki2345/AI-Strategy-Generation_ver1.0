# 访问策略表单字段规范

> 与紫鸟平台「新增访问策略」表单 1:1 对齐

---

## 1. 主表单字段

### 策略名称 (`policy_name`)
- **类型**：text
- **必填**：是
- **最大长度**：24 字符
- **说明**：策略用途的简洁描述

### 生效成员 (`effective_members`)
- **类型**：selector
- **必填**：是
- **选项**：`所有成员` | `除BOSS外所有成员` | `指定成员`
- 选择「指定成员」时，需要提供 `custom_members`（在成员/部门/角色 tab 中勾选）

### 生效平台账号 (`effective_accounts`)
- **类型**：selector
- **必填**：是
- **选项**：`所有账号` | `指定账号`
- 选择「指定账号」时，需要提供 `custom_accounts`

### 生效时段 (`effective_period`)
- **类型**：compound
- **必填**：是
- **模式选项**：`一直生效` | `指定时段生效`

当模式为「指定时段生效」时，包含以下子字段：

| 子字段 | 字段 ID | 类型 | 值 |
|---|---|---|---|
| 生效日期 | `effective_date_range` | date_range | `start_date` / `end_date` |
| 生效周期 | `effective_weekdays` | weekday_selector | 一/二/三/四/五/六/七 |
| 生效时间 | `effective_time_range` | time_range | `start_time` / `end_time` |

### 访问策略描述 (`policy_description`)
- **类型**：textarea
- **必填**：否
- **最大长度**：400 字符

### 访问审批限制 (`approval_restrictions`)
- **类型**：checkbox_group
- **独立勾选项**：
  - `仅boss账号可审批`
  - `不允许申请访问`

## 2. tab_指定网页生效策略 (`specific_page_rules`)

每条 URL 规则定义一个 URL 模式、访问模式和可选的操作行为控制。

### URL 语法规则

| 功能 | 符号 | 说明 |
|---|---|---|
| 协议 | `https://` | 每条 URL 必须以此开头 |
| 通配符 | `*` | 跨子域名/顶级域模糊匹配。如 `https://*.amazon.*/orders-v3` 匹配所有亚马逊站点 |
| 终止符 | `$` | 仅匹配主域名，所有子路径被阻断。如 `https://sellercentral-japan.amazon.com/$`。结合 `[^/]*` 匹配任意非斜杠字符 |
| 查询参数 | `?` | `preserve_params: false`（默认）丢弃 `?` 后内容。`preserve_params: true` 要求参数完全一致 |

**最佳实践**：使用 `*` 通配符一条规则覆盖多个站点，不要逐站点列多条。

### 规则模板

```
url:             <URL 模式>
preserve_params: true | false (默认 false)
mode:            可访问 | 已限制
```

- **可访问**：页面可访问，可通过 behavior_controls 限制具体操作
- **已限制**：页面完全阻断，不需要 behavior_controls

### behavior_controls（仅在 mode = 可访问 时生效）

| 控制项 | 选项 | 说明 |
|---|---|---|
| 复制 | `允许` / `允许且记录` / `限制且记录` | 浏览器环境中的复制操作 |
| 文件上传 | `允许` / `允许且记录` / `限制且记录` | 文件上传操作 |
| 文件下载 | `允许` / `允许且记录` / `限制且记录` | 文件下载操作 |
| 打印 | `允许` / `允许且记录` / `限制且记录` | 打印操作 |
| 鼠标点击 | `允许` / `限制` | 鼠标点击；限制后页面变为只读 |
| 键盘输入 | `允许` / `限制` | 文本字段的键盘输入 |
| 查看密码框 | `允许` / `限制` | 密码/敏感字段的明文查看 |

## 3. tab_所有网页通用策略 (`global_page_rules`)

作用于账号环境下**所有页面**的全局行为规则。与上面相同的 7 项控制、相同的选项：

| 控制项 | 选项 |
|---|---|
| 复制 | `允许` / `允许且记录` / `限制且记录` |
| 文件上传 | `允许` / `允许且记录` / `限制且记录` |
| 文件下载 | `允许` / `允许且记录` / `限制且记录` |
| 打印 | `允许` / `允许且记录` / `限制且记录` |
| 鼠标点击 | `允许` / `限制` |
| 键盘输入 | `允许` / `限制` |
| 查看密码框 | `允许` / `限制` |

注意：前 4 项有三档选项（允许 / 允许且记录 / 限制且记录），后 3 项只有二档（允许 / 限制）。
