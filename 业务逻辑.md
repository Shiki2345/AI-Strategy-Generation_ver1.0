# Agent 调用 Skill 的业务逻辑

## 整体流程（5个环节）

```
用户消息
   │
   ▼
① Agent 收到消息
   │
   ▼
② 读取 system-reminder（Skill 目录）
   │         ← 相当于翻开"说明书目录"
   ▼
③ 匹配 TRIGGER 条件
   │         ← 相当于找到对应那本说明书
   ▼
④ 调用 Skill 工具 → Skill 展开为完整指令
   │         ← 相当于打开说明书第一页
   ▼
⑤ Agent 按照 Skill 内的步骤逐一执行
              ← 相当于按步骤拼积木
```

---

## 关键机制：谁负责什么

| 角色 | 职责 | 类比 |
|---|---|---|
| **Agent（Claude）** | 识别意图、调用 Skill、执行步骤 | 拼积木的人 |
| **Skill 工具** | 把 Skill 名称展开为完整提示词 | 说明书本身 |
| **system-reminder** | 告知 Agent 有哪些 Skill 可用 | 积木盒上印的套装目录 |

> **核心要点**：Skill 不是独立运行的程序，它是一段被注入到 Agent 上下文里的**超长指令**。Agent 执行的还是自己的工具（Read/Edit/Bash），只是遵照 Skill 给出的步骤。

---

## 具体例子：访问策略 Skill

以 `skill_definition.json` 为蓝本，走一遍完整调用链。

### 第 0 步：系统启动时，Skill 目录被加载进 system-reminder

```
Available skills:
- access_policy_builder: 通过对话生成网页访问策略配置
  TRIGGER when: 用户说"禁止访问"、"限制网站"、"上班时间"等关键词
```

### 第 1 步：用户发消息

```
用户：我想在上班时间禁止员工访问视频网站
```

### 第 2 步：Agent 扫描 system-reminder，命中 TRIGGER

```
检测到关键词：
  "上班时间" ✓  →  触发条件匹配
  "禁止"     ✓
  "视频网站" ✓

→ 匹配到 Skill: access_policy_builder
```

### 第 3 步：Agent 调用 Skill 工具

```python
Skill({ skill: "access_policy_builder" })
```

Skill 工具把 `skill_definition.json` 里定义的 `dialogue_flow` 展开成完整提示词，注入 Agent 上下文。

### 第 4 步：Agent 按 Skill 内的步骤执行（5个阶段）

```
phase_1: 识别意图
  → 回复："明白了！你想配置一个时间限制 + 黑名单的组合策略。"

phase_2: 收集基本参数（多轮对话）
  → 问：工作时间是什么时候？
  ← 用户：周一到周五 9:00-18:00
  → 问：需要屏蔽哪些网站？
  ← 用户：B站、爱奇艺、YouTube

phase_3: 确认细节（Skill 内置的领域知识）
  → 检测到 "qq.com" → 触发警告：
    "qq.com 会影响 QQ 邮箱，建议改用 v.qq.com"

phase_4: 生成 JSON 配置
  → 读取 config_template.json，填入参数
  → 输出完整策略 JSON

phase_5: 交付 + 后续引导
  → "配置已生成，是否直接应用？"
```

### 第 5 步：Skill 内的领域知识发挥作用

phase_3 里的 `domain_precision_check` 硬编码了领域专家经验：

```json
"warning_domains": ["qq.com", "baidu.com"],
"suggestions": {
  "qq.com": "v.qq.com（腾讯视频）"
}
```

这是普通 Agent 不会主动做的事——**Skill 把领域专家经验固化成了可复用的规则**。

---

## 一句话总结

```
用户意图 → TRIGGER 匹配 → Skill 工具展开 → Agent 获得"专家级步骤" → 按步骤执行工具
```

Skill 的价值不是"让 Agent 变聪明"，而是把**某个领域的最佳实践**打包成说明书，让 Agent 每次面对同类问题时都能按专家水准执行，而不是每次靠自己即兴发挥。
