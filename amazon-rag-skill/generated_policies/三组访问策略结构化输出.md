## 策略一：6组Case数据隔离策略

### 基本信息

- **策略ID**：`case_data_isolation_group6_001`
- **策略名称**：6组Case数据隔离策略
- **策略类型**：`data_ownership_isolation`（数据所有权隔离）
- **风险等级**：Medium
- **版本**：1.0
- **创建时间**：2026-04-13

### 受控资源

- **资源类型**：Case 工单记录
- **控制层级**：行级别安全（Row-Level Security）
- **敏感级别**：Medium

### 可见性规则

#### 规则 1：6组成员只能查看本人创建的Case

- **规则ID**：`rule_01`
- **优先级**：1
- **主体**：`group_6`（6组）
- **条件**：`created_by = {{current_user_id}}`
- **动作**：Allow
- **说明**：6组成员登录后，Case列表仅展示由本人创建的记录

#### 规则 2：其他组成员不可查看6组的Case

- **规则ID**：`rule_02`
- **优先级**：2
- **主体**：非6组所有成员
- **条件**：`owner_group = group_6`
- **动作**：Deny
- **拒绝提示**：你无权查看该Case，此记录属于其他组
- **说明**：非6组用户查询时，系统自动过滤掉所有 owner_group = group_6 的记录

### 数据过滤逻辑

**6组成员查询时注入：**

```sql
WHERE created_by = :current_user_id AND owner_group = 'group_6'
```

**非6组成员查询时注入：**

```sql
WHERE owner_group != 'group_6'
```

### 适用范围

- **受限对象**：6组全体成员
- **例外放行**：
  - 角色：系统管理员、超级管理员
  - 组：管理层
  - 说明：用于运营审计，可查看所有Case

### 控制动作覆盖

- **列表页浏览**：静默过滤，不展示无权限记录，不提示存在被隐藏的数据
- **详情页访问**：提示"无权访问"，提供"返回列表"按钮
- **搜索**：搜索结果自动排除无权限的Case
- **导出**：仅导出有查看权限的Case

### 执行与审计

- **严格模式**：开启
- **允许绕过**：否
- **记录日志**：是
- **日志留存**：90 天
- **记录事件**：
  - 越权访问尝试（`unauthorized_access_attempt`）
  - 过滤器命中（`data_filter_applied`）
  - Case查看（`case_viewed`）
  - Case搜索（`case_searched`）
  - Case导出（`case_exported`）
