# 访问策略 - 表单填写版结构化输出

> 严格对齐平台「新增访问策略」表单字段，每个字段直接可填入

---

## 策略一：6组Case数据隔离策略

> **平台适配性判断：不适用**
>
> 该策略的核心是数据库行级别的记录隔离（按 Case 创建者过滤），
> 而本平台的能力边界是"网页 URL 访问控制 + 浏览器行为限制"，
> 无法实现按数据记录所有权进行过滤。
>
> **建议实现方式：** 在业务系统后端 / 数据库层实现 Row-Level Security，
> 或在 Case 管理系统的权限模块中配置，不通过本平台处理。

## 策略二：买家号Checkout结账页屏蔽策略（v1.0 全天屏蔽）

### 左侧 - 基本信息

| 表单字段 | 填写内容 |
|---------|---------|
| **策略名称** | 买家号Checkout结账屏蔽 |
| **生效成员** | 除BOSS外所有成员 |
| **生效平台账号** | 所有账号 |
| **生效时段** | 指定时段生效 |
| ↳ 生效日期 | 不勾选 |
| ↳ 生效周期 | 勾选，选中：一 二 三 四 五 六 七（全选） |
| ↳ 生效时间 | 勾选，开始时间 `00:00` → 结束时间 `23:59` |
| **访问策略描述** | 屏蔽买家账号Secure Checkout结账页面，防止人员恶意或误操作下单导致账号资金损失、触发平台风控锁定账号。拦截范围包括checkout结账页、购物车页。 |
| **访问审批限制** | |
| ↳ 仅boss账号可审批 | 勾选 |
| ↳ 不允许申请访问 | 勾选 |

### 左侧Tab - 指定网页生效策略

| 序号 | 要屏蔽的 URL 规则 | 说明 |
|------|-----------------|------|
| 1 | `www.amazon.com/checkout/*` | Secure Checkout 结账页（已验证） |
| 2 | `www.amazon.com/gp/cart/*` | 购物车页面 |
| 3 | `www.amazon.com/gp/buy/spc*` | 旧版结账页（兜底） |

### 右侧Tab - 所有网页通用策略

| 操作项 | 设置 |
|-------|------|
| 复制 | 允许且记录 |
| 文件上传 | 允许且记录 |
| 文件下载 | 允许且记录 |
| 打印 | 允许且记录 |
| 开发者模式 | 允许且记录 |
| 查看密码框 | 限制 |
