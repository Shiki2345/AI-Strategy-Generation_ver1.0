# 项目文件关系说明

整个项目有两代文件，展示了从**概念探索**到**生产落地**的演进过程。

---

## 第一代：`体验内容/`（概念探索期）

以"时间黑名单"场景为例的 Skill 原型系统：

```
skill.md                          ← 最早期的概念草稿（极简版 Skill 定义）
                                     定义了 4 步 workflow，仅是思路记录

access_policy_skill_package/
├── skill_definition.json         ← 完整 Skill 定义（对话流程 5 阶段）
│                                    包含：触发模式、每步问题、确认逻辑
├── config_template.json          ← 策略配置的模板（带 {{PLACEHOLDER}} 占位符）
│                                    填充后变成最终的策略 JSON
├── presets.json                  ← 预设选项库
│                                    （国内视频/国际视频/短视频/社交/游戏/购物）
└── example_output.json           ← 模板填充后的示例输出

工作时间视频网站限制.json            ← 实际生成的策略文件（内容与 example_output 相同）
                                     这就是一次对话的最终产物
```

**关系**：`skill_definition` 控制对话流程 → 从 `presets` 取预设选项 → 把参数填入 `config_template` → 输出策略 JSON

---

## 第二代：`amazon-rag-skill/`（生产落地期）

在第一代基础上增加了 **RAG 检索** 和 **Amazon 专属场景**：

```
data/amazon_urls.json             ← RAG 知识库（20条 Amazon Seller Central 页面）
rag_simple_db.json                ← 知识库的简化版（difflib 版本使用）

skills/
└── approval_based_access_control_skill.json  ← 第二代成熟 Skill
                                                 含 RAG 相似度阈值（≥0.8/0.5~0.8/<0.5）
                                                 含 6 阶段对话流

generated_policies/               ← 所有实际生成的策略输出
├── 运营专员账户详情页审批策略.json     ← Session 1：Skill 首次炼制的产物
├── 实习生定价管理审批策略.json         ← Session 2：Skill 复用的产物（仅 2 轮对话）
├── 客服专员账户详情页完全禁止策略.json
├── 财务专员多页面访问限制策略.json     ← 财务专员场景 v1
└── 财务专员多页面访问审批策略_v2.json  ← 财务专员场景 v2（最终版）

skill_finalization_report.json    ← Skill 炼制过程记录
财务专员多页面访问限制_skill_report.json  ← 财务专员 Skill 报告
skill_reuse_comparison_report.json ← 首次配置 vs Skill 复用的效率对比
skill_reuse_deployment.json       ← Skill 部署配置
```

---

## 核心演进逻辑

```
skill.md（概念）
    ↓ 丰富对话流程 + 加入预设库
access_policy_skill_package/（原型）
    ↓ 替换 difflib → RAG 检索
    ↓ 替换通用场景 → Amazon 专属 URL
    ↓ 加入 Skill 炼制 + 复用机制
amazon-rag-skill/skills/（生产 Skill）
    ↓ 每次对话产出
generated_policies/（策略文件）
    ↓ 沉淀复用经验
*_report.json（效果记录）
```

---

## 一句话总结

`skill.md` 是最初的思路草稿，`access_policy_skill_package` 是第一个可运行的 Skill 包（含模板+预设），`amazon-rag-skill/` 是加入 RAG 后的生产版本，`generated_policies/` 是每次对话的输出物，`*_report.json` 是系统学习和效果验证的记录。
